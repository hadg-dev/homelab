# Homelab Project Guidelines

## Project Overview
This is a Kubernetes GitOps homelab infrastructure setup with automated bootstrap/destroy workflows, GitOps configuration management (ArgoCD, Keycloak), and monitoring stacks (Prometheus, Grafana).

## Python Code Style
All Python code must adhere to:
- **Formatter**: [Black](https://github.com/psf/black) — run `black homelab/scripts/`
- **Import sorting**: [isort](https://pycqa.github.io/isort/) — run `isort homelab/scripts/`
- **Type checking**: [mypy](http://mypy-lang.org/) — run `mypy homelab/scripts/`

Before proposing Python changes:
1. Ensure code is Black-formatted (88-char line length)
2. Organize imports with isort (Black-compatible profile)
3. Add type hints and verify with mypy (strict mode preferred)
4. Use Python 3.12+ syntax where appropriate

Reference config:
```toml
[tool.black]
line-length = 88

[tool.isort]
profile = "black"

[tool.mypy]
python_version = "3.12"
strict = true
```

## Shell Scripting
- All shell scripts must be **zsh-compatible** (not bash)
- No non-standard shell extensions or assumptions
- Prefer explicit path references; avoid relying on $PATH for homelab-specific tools

## Kubernetes & YAML
- Keep manifests **declarative**; do not add cloud-provider-specific resources (AWS, Azure, GCP)
- Preserve **comments and formatting** in YAML files when editing
- Use Helm values for customization; avoid duplicating manifests
- Reference ingress patterns from `k8s/ingress/` for new routes
- TLS certificates handled via `k8s/security/homelab-tls.yaml`

## Helper Functions
- Prefer using existing helpers in `homelab/scripts/` over writing new utilities
- Common tasks covered by: `bootstrap.py`, `ingress.py`, `platform.py`, `state.py`, `tls.py`
- Check `state.py` for state management before adding new logic

## Security & Secrets
- **Never hardcode secrets or credentials** in configuration files
- Use environment variables or external secret stores
- Do not commit sensitive data to git

## Logging
Use the centralized logger from `homelab/scripts/logger.py` for all Python scripts

## Do Not Change
- `broken-pod.yaml` — for debugging/reference only
- `state.json` — managed by automation; read-only unless explicitly updating state
- Core bootstrap/destroy logic without thorough testing

## Architecture Notes
- **Bootstrap**: Orchestrated via `bootstrap_homelab.zsh` → `scripts/bootstrap.py`
- **GitOps**: ArgoCD syncs from `gitops/platform/`
- **Monitoring**: Prometheus scrapes, Grafana visualizes
- **State tracking**: Stored in `state.json` for idempotency
